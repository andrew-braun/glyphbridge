# Migration to Noto Sans Font

Switch the site's primary typography to Noto Sans to improve script support.

## Current State

- Fonts are defined in `src/lib/styles/_variables.scss`.
- `global.scss` handles base styling but currently lacks `@font-face` declarations for local files.
- Some components (`GuestHomeShell.svelte`) have hardcoded font stacks.
- Font files are located in `src/lib/assets/fonts/`.

## Implementation Plan

### 1. Configure Font Loading

Add `@font-face` declarations to `src/lib/styles/global.scss` to load the variable font files.

- Load `NotoSans-VariableFont_wdth,wght.woff2` as "Noto Sans".
- Load `NotoSans-Italic-VariableFont_wdth,wght.woff2` as "Noto Sans" (italic).

### 2. Update Design Tokens

Modify `src/lib/styles/_variables.scss` to update the font stacks:

- Update `$font-body` to prioritize `"Noto Sans"`.
- Update `$font-thai` to use `"Noto Sans"`.
- Update `$font-display` to prioritize `"Noto Sans"` if appropriate, or keep as is if the brand identity requires "Chakra Petch". (Recommendation: update to Noto Sans for consistency).

### 3. Clean up Hardcoded Font-Families

Find and replace hardcoded font stacks in components to use the design tokens.

- Update `src/lib/components/content/home/GuestHomeShell.svelte` (line 234) to use `$font-thai` instead of `"Thai", "Arial Unicode MS", sans-serif`.

### 4. Verification

- Run `pnpm check` to ensure no SCSS syntax errors.
- Manually verify font rendering in browser.

# Remove page-intro sections, preserve progress data

## Context

All five top-level pages have a `<section>` hero block using `.page-intro__*` BEM classes at the top. The goal is to remove these hero sections entirely ("show, don't tell"), while keeping any user-specific progress stats visible on-page.

The `MainNav` already displays `{$knownLetters.length}` and `{$knownWords.length}` as counter badges on the Letters and Words nav links, so those counts are visible at all times.

## Pages and their page-intro content

| Page                      | Has user progress data? | What data?                                                                                                                                            |
| ------------------------- | ----------------------- | ----------------------------------------------------------------------------------------------------------------------------------------------------- |
| `alphabet/+page.svelte`   | Yes                     | `{$knownLetters.length}` unlocked, `{allLetters.length}` taught                                                                                       |
| `words/+page.svelte`      | Yes                     | `{$knownWords.length}` (shown twice as badges)                                                                                                        |
| `practice/+page.svelte`   | No                      | Descriptive badges only ("randomized drills", "10-question sprint", "built from completed lessons"). Stats already shown in start-state metric pills. |
| `learn/+page.svelte`      | No                      | `{lessons.length}` is a static publication count, not user-specific                                                                                   |
| `auth/+page.svelte`       | No                      | Descriptive badges only ("one-time email code", "local-first shell", "optional account")                                                              |
| `learn/[id]/+page.svelte` | N/A                     | No page-intro; uses its own step UI                                                                                                                   |

## Plan

### 1. `alphabet/+page.svelte` — Remove hero, Progress bar already preserves data

Remove lines 78-85 (the `<section>` with page-intro elements). The `<Progress>` component at lines 88-95 already shows the same letter-count data in a better visualization.

### 2. `words/+page.svelte` — Remove hero, add lightweight count header

Remove lines 26-33 (the `<section>` with page-intro). Add a small line above or inside the word grid showing the word count — something unobtrusive like `"{$knownWords.length} words collected"` as a `<p>` or `<h2>` — not a hero, just a lightweight context header.

### 3. `practice/+page.svelte` — Remove hero entirely

Remove lines 93-101 (the `<section>` with page-intro). No progress data to preserve — the start-state metric pills at lines 127-133 already show `{$knownLetters.length}` and `{$knownWords.length}`.

### 4. `learn/+page.svelte` — Remove hero entirely

Remove lines 69-77 (the `<section>` with page-intro). No user-specific data to preserve.

### 5. `auth/+page.svelte` — Remove hero entirely

Remove lines 84-98 (the `<section>` with page-intro). No user-specific progress data.

### 6. Clean up SCSS

- Remove the `.page-intro` class and all its `&__*` nested styles from `src/lib/styles/global.scss` (~lines 249-290)
- Remove the dead `&__intro` style rules from each page's `<style>` block (the `margin-bottom: $space-sm` rules that referenced the now-deleted elements)

### 7. Verify

Run `pnpm check` to confirm no errors.

## On the sidebar question

A sidebar is unnecessary here. The `MainNav` already surfaces letter/word counts globally in its navigation pill badges. Each page preserves its meaningful progress data inline (Progress bar on alphabet, count header on words, metric pills on practice start screen). Adding a sidebar would duplicate what the nav already provides without adding new information.
